# World Cup Oracle — מסמך אפיון

**גרסה:** 2.0 (פורמט 48 קבוצות + כיוון עיצובי מאושר)
**תאריך:** יוני 2026
**בעלים:** Gal Tayeb
**מטרת המסמך:** להגדיר מאפס מה המוצר, למה הוא קיים, ומה צריך להיבנות — כבסיס שאפשר למסור ל-Claude Code (ראו `TASKS.md` לצד מסמך זה).

---

## 1. מה המוצר (במשפט אחד)

**World Cup Oracle** הוא סימולטור מונדיאל 2026 מבוסס AI: המשתמש לוחץ כפתור אחד, המערכת מריצה את כל הטורניר — 12 בתים, שלב 32, ועד הגמר — ומציגה bracket אינטראקטיבי עם הסתברות זכייה לכל קבוצה. לחיצה על משחק פותחת הסבר עומק: למה המודל חזה את התוצאה.

המוצר עונה על שאלה אחת: **"מי יזכה במונדיאל לפי המודל — ולמה?"**

---

## 2. למה זה (הבעיה עם הגרסה הקודמת)

הגרסה הישנה ניסתה להיות שלושה מוצרים בו-זמנית: כלי תחזית, משחק ניחושים, ומערכת משתמשים/קבוצות. אף אחד לא הושלם והם סתרו זה את זה:
- **לוח התוצאות שיקרי** — הקוד שמר `actual_result = predicted_label`, כך שה-AI "צדק" תמיד.
- **אין מקור תוצאות אמת** — אי אפשר לדעת מי ניחש נכון.
- **קבוצות ללא משמעות** — שורות ריקות, בלי מטרה.

ההחלטה: למקד הכל סביב הסימולטור. כל מה שלא משרת אותו יורד מה-UI.

## 4. פורמט מונדיאל 2026 (קריטי לסימולציה)

- **48 קבוצות**, 12 בתים (A–L), 4 בכל בית, round-robin (3 משחקים לכל קבוצה).
- **עולים לנוקאאוט (שלב 32):** 12 מנצחי בתים + 12 סגניות + **8 הקבוצות השלישיות הטובות ביותר** = 32.
- **שלבי נוקאאוט:** שלב 32 → שמינית → רבע → חצי → (משחק מקום 3) → גמר.
- האלופה משחקת 8 משחקים. סה"כ 104 משחקים.

### 4.1 שוברי שוויון בבית (לפי הסדר)
1. הפרש שערים
2. שערים שהובקעו
3. ראש-בראש (נקודות, הפרש, שערים בין השוות)
4. פייר-פליי (כרטיסים)
5. הגרלה

> הסימולציה חייבת לדרג קבוצות בבית לפי הסדר הזה, לא רק נקודות. (פייר-פליי לא ניתן לחיזוי — אפשר לדלג עליו או להחליף בהגרלה אקראית כ-fallback.)

### 4.2 "8 השלישיות הטובות" (החלק המורכב)
- לאסוף את 12 הקבוצות שסיימו 3 בכל בית.
- לדרג אותן ביניהן (נקודות → הפרש שערים → שערים → ...).
- לקחת את 8 הטובות.
- לשבץ אותן למנצחי בתים לפי **טבלת השיבוץ הרשמית של פיפ"א** (תלוי אילו בתים תרמו שלישיות — צריך למפות).

## 5. תכולת שלב ראשון (MVP)

### 5.1 מסך ראשי — הסימולטור
- כפתור מרכזי אחד: **"Run simulation"**.
- מצב טעינה אנימטיבי בזמן הרצה.
- שלושה stat cards: אלוף חזוי / הסתברות זכייה / דיוק המודל.

### 5.2 תצוגת Bracket
- שלב הבתים: 12 בתים, מי עלה מכל אחד (+ סימון 8 השלישיות).
- נוקאאוט: שלב 32 → שמינית → רבע → חצי → גמר, כעץ חזותי.
- האלופה מודגשת. ליד כל קבוצה — הסתברות.

### 5.3 הסבר משחק (Drill-down)
- לחיצה על משחק → פאנל עם הסתברות התוצאה, הפיצ'רים שהשפיעו (דירוג, xG, צורה, head-to-head), ומשפט הסבר בשפה פשוטה.

### מחוץ ל-MVP (Backlog)
- התחברות / פרופיל / leaderboard ניחושים.
- שמירת סימולציות.
- תוצאות אמת בזמן אמת.
- מצב "מה-אם" (שינוי תוצאה ידני).

## 6. שיפור המודל (פרויקט נפרד, מתבצע ב-Colab)

### 6.1 בעיות במודל הנוכחי
1. **דיוק 46.7%** על test — גרוע מבייסליין "תמיד Home Win" (~50%).
2. **מאומן על 281 משחקים בלבד**; `results.csv` (~48K) לא בשימוש.
3. **דליפת נתונים חמורה** — `group_stats.csv` הם נתוני מונדיאל 2022 *עצמו* (xG, ניצחונות בטורניר). המודל משתמש בתוצאות כדי לחזות אותן.
4. **NaN בתוויות** נכנס ל-LabelEncoder.
5. **פיצ'רים חזקים לא מנוצלים:** צורה אחרונה, head-to-head, מגרש ניטרלי, ממוצע שערים, נקודות פיפ"א.

### 6.2 כיוון התיקון
- לאמן על `results.csv` המלא, לא רק 32 קבוצות.
- **אפס דליפה:** כל פיצ'ר מחושב רק ממידע שהיה זמין לפני המשחק (דירוג לפי תאריך, צורה מ-N משחקים אחרונים, h2h היסטורי).
- פיצ'רים חדשים: הפרש נקודות פיפ"א, צורה (5–10 משחקים), ממוצע שערים, h2h, אינדיקטור מגרש ניטרלי.
- מודל: השוואת XGBoost / GradientBoosting / RandomForest עם **time-based split** (לא רנדומלי — למנוע דליפת עתיד).
- יעד דיוק ריאלי: 52–58%.
- **`predict_proba`** במקום `predict` בלבד — להזין אחוזי bracket והסבר.

## 7. ארכיטקטורה טכנית

### 7.1 קיים (נשמר)
- **Backend:** Django + DRF, JWT (SimpleJWT), CORS פתוח, SQLite.
- **Frontend:** React 19, React Router 7, Axios. **אין ספריית UI, אין Tailwind** — CSS גלובלי + inline styles.
- **מודל:** נטען ב-`predictor/views.py` דרך joblib.

### 7.2 שינויים בבאק
- **Endpoint חדש:** `POST /api/simulate/` — מריץ את כל הטורניר, מחזיר JSON מובנה (בתים, שלב 32, נוקאאוט, הסתברויות). לוגיקת הסימולציה עוברת מהנוטבוק לשירות בבאק (`predictor/simulation.py`).
- **תיקון באג:** הסרת `actual_result = predicted_label` ב-`save_prediction`.
- **endpoints שיורדים מה-UI (נשמרים בקוד):** register, login, leaderboard, group-leaderboard, my_predictions, user/profile.

## 8. עיצוב (מאושר)

### 8.1 בעיות בעיצוב הנוכחי
- כחול `#007bff` גנרי חוזר בכל מקום, צבעי Bootstrap בלי Bootstrap.
- inline styles מפוזרים ב-`App.js`, אין שפה עיצובית.
- טופס + 3 טבלאות מתחרים במסך אחד.

### 8.2 כיוון מאושר
- **Design tokens:** קובץ CSS מרכזי עם משתנים (צבעים, טיפוגרפיה, מרווחים, radius).
- **פלטה:** ירוק דשא עמוק (`#0F6E56`) כצבע פעולה ראשי; **זהב (`#BA7517`) שמור לגמר ולגביע בלבד** (מסמן ניצחון); ניטרלים כהים.
- **מסך ראשי:** hero אחד — כפתור סימולציה גדול במרכז + 3 stat cards. ה-bracket הוא הגיבור הויזואלי.
- **Bracket:** הקבוצה המנצחת מודגשת ברקע ירוק בהיר, המפסידה דהויה; אחוז לצד כל קבוצה; לחיצה פותחת "למה".
- **דגלים אמיתיים:** ספריית `flag-icons` (לא emoji) — שדרוג ניכר.
- **מיקרו-אינטראקציות:** אנימציית טעינה לסימולציה, hover על משחקים, חשיפה הדרגתית של שלבים.
